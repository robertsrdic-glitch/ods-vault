# NorthStar — Self Critique

## Direction A+

- Six changes were introduced, more than a true "final 5%" implies. In practice only two are real identity marks: the **Certainty Arc** and the **single-sharp-corner rule**. The figure/loading/feedback treatments are refinements, not signatures.
- If forced to keep only one signature element, keep the Certainty Arc — it is both a brand mark and a functional honesty device (the AI Rules require stating uncertainty), so it earns its place rather than decorating.
- Confidence: 7/10 that the Arc alone becomes memorable at scale. 5/10 that all six changes together still read as one coherent direction rather than a list of details — worth testing the Arc in isolation before committing to the full set.

## Open risks flagged during the sprint

- **Certainty Arc** could be misread as a progress/percentage-complete indicator if shown unlabelled — it must always pair with text (consistent with "icons never replace text").
- **Sharp-corner rule** needs one consistent corner position system-wide, or it will read as an inconsistency/bug rather than a mark.
- **1a (Calm Intelligence)** overall risk: can read as sparse/unfinished if spacing/type discipline slips — the direction depends on a strong system to stay legible as calm rather than empty.

## Known gap

No Design Tokens architecture is defined in any source document. This was flagged rather than invented, per the Approved Design Constraints — a real tokens system (global → semantic → component tiers) should be scoped as its own decision once component work begins, not backfilled here.
