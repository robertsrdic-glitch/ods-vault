# Purpose

This document is an index of the named workflows supported by the AI Product OS. It does not describe how the organization operates — that is defined once, in full, by the Operating Model. This document exists so that anyone can see, at a glance, which workflows exist and where each one is fully defined.

---

# Workflow Categories

- **Strategic Workflows** — validating direction before commitment.
- **Product Workflows** — defining what is being built.
- **Design Workflows** — defining how it should feel and behave.
- **Engineering Workflows** — building it.
- **Quality Workflows** — verifying it before approval.
- **Release Workflows** — making approved work current.
- **Governance Workflows** — changing the AI Product OS itself.

# Relationship to Operating Model

The Operating Model is the single source of truth for how the organization operates: stage purposes, owner roles, outputs, exit criteria, decision gates, and the review and decision models all live there and only there. This document never restates that content. Where this index and the Operating Model appear to disagree, the Operating Model governs.

---

# Workflow Ownership

Each workflow listed above is owned by the same role the Operating Model assigns as Primary Owner for its corresponding lifecycle stage or gate. This document does not assign ownership independently — it points to where ownership is defined.

---

# Workflow Documentation Rules

- A workflow is added to this index only after it is fully defined elsewhere (Operating Model, or a dedicated governance document such as the RFC Process).
- No entry in this index may describe stage mechanics, ownership detail, or approval criteria — those belong solely to the workflow's defining document.
- Adding, removing, or re-categorizing a workflow in this index follows the same governance as any other change to a document referencing a Foundation Document.

---

# Definition of Success

This index succeeds when it remains a short, accurate map of what workflows exist and where each is defined — and never drifts into re-describing how the organization operates.
