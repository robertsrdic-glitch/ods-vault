# Repository Standard

Version: 1.0
Status: Draft
Owner: Founder

---

# Git as Source of Truth

The repository, not any individual's memory or a prior conversation, is the authoritative record of product knowledge and decisions. When documentation and assumption conflict, documentation wins.

---

# Repository Rules

- No artifact is deleted. Superseded material moves to Archive.
- Every document declares an owning role and a status.
- Structural changes (new folders, restructuring) require Founder approval.
- Documentation is written before implementation begins, not after.

---

# Artifact Ownership

Every file in the repository has exactly one owning role, as defined in the Organization Charter. Ownership is stated on the artifact. An artifact without a stated owner is incomplete.

---

# Naming Rules

- File names are descriptive and unique within their folder.
- No duplicate file extensions or ambiguous naming.
- Names do not change without updating every reference to that artifact.

---

# Review Policy

- Every artifact begins as Draft.
- Only the owning role, or the Founder, may mark an artifact Approved.
- No role modifies another role's Approved artifact without that role's involvement.
- Conflicts discovered during review are reported, not silently resolved.
